### A3 核心技术结论 · **pass（根因）/ partial（探针声明）**

#### 3.1 根因补丁 `48b5bae` · **pass**

`git show 48b5bae -- packages/runtime/scripts/injector.mjs` 证实：

- 旧：仅 `palette.accent`
- 新：循环 `["accent","secondary","surface","text"]` + CSS color 正则校验
- `loadCatalogMember` 同步扩展

现行 `injector.mjs:380-389` 与 commit 一致。

#### 3.2 renderer surfaceLuma · **pass（逻辑存在）+ 边界 caveat**

`renderer-inject.js:105-142` 读取四色并算 `surfaceLuma`；`resolveAppearance`（:358-367）阈值 0.45/0.62 存在。

**边界**：`surfaceLuma` **仅对 `#rrggbb` 6 位 hex** 计算：

```js
if (requestedSurface && /^#[\da-f]{6}$/i.test(requestedSurface)) { … }
```

`oklab(...)` / `#rgb` / `rgb(...)` 的 surface → `surfaceLuma=null` → 回落 `detectShellAppearance()`。  
当前内置主题用 hex，本机 dark 主题安全；**schema 允许多种 CSS color 时仍可能回归**。

#### 3.3 闪白因果链 · **pass（逻辑自洽）**

报告链路：

```text
loadTheme 只透传 accent
  → palette.surface = undefined
  → surfaceLuma 不进入 number 分支（或 NaN）
  → resolveAppearance 走 shell auto
  → 暗色主题可挂 dream-theme-light
  → 开项目闪白
```

与源码一致。`typeof config.surfaceLuma === "number"` 对 `NaN` 为 true，但 `NaN <= 0.45` / `>= 0.62` 皆 false → 仍走 shell auto。报告写「NaN → else」方向正确。

#### 3.4 probe-white-flash · **pass（非假阳性）**

- 默认 `pass: false`（:139）
- CDP 不可达：`fetch` / `no app:// page` throw → `process.exitCode = 1`
- 断言失败：`failed.length > 0` → `pass=false` → `exitCode = 2`
- 本机历史结果：`pass=true`，`surfaceLuma=0.105…`，`bodyLuma≈0.185`，`darkClass=true`，`failed=[]`
- CDP live 复证：`http://127.0.0.1:9335/json/list` → 200

**非假阳性**。上一轮 pass 声明可采信。

`scripts/windows/probe-project-hd.mjs`：
